灰度图

buffer

I2C / SPI

OLED

PWM
通过改变方波占空比调节 LED 平均亮度。

pthread
POSIX 线程库，在 Linux 上创建和管理多个并发执行的线程。
batch
一次送入模型处理的多张图像组成的一组。
生产者–消费者
一个线程往队列放入数据，另一个线程取出数据并处理。
互斥锁
保证同一时刻只有一个线程读写共享数据，避免数据错乱。
条件变量
队列空或满时让线程等待，有数据可用时再唤醒。

CNN
用卷积层从图像中提取特征并做分类的神经网络。
LeNet / Conv / Pool / FC
一种小型 CNN 结构；卷积层提取局部特征；池化层缩小尺寸；全连接层输出各类得分。
SGD
随机梯度下降，每次用一小批样本计算梯度并更新权重。
fp32
32 位浮点数，训练阶段权重常用的数值格式。
weights.bin
导出的二进制权重文件。
golden
用固定输入跑模型得到的参考输出，用来核对后续推理是否正确。
checkpoint
训练中途保存的权重快照，可从中恢复或挑选最佳轮次。

ch09 模型训练与部署篇 9.1 浮点 vs int8：体积、带宽、延迟动机 PTQ / 剪枝 / QAT / 蒸馏；产出 int8 部署包 per-tensor PTQ；per-channel PTQ；剪枝；对比表 int8 weights.h + 精度–体积–延迟表
int8
8 位有符号整数（−128 到 127），量化后常用的权重存储格式。
PTQ
训练完成后，把 fp32 权重量化为 int8 的后训练量化。
scale / zero-point
浮点数与 int8 互转时的比例系数；zero-point 是浮点 0 对应的整数值。
剪枝
删除部分权重或通道，使模型体积变小、计算量减少。
QAT
在训练过程中模拟 int8 运算，提高量化后的精度。
蒸馏
用 fp32 模型的输出指导更小的 int8 模型学习。
per-tensor / per-channel
整张权重共用一个 scale；或每个输出通道各用一个 scale。

ch10 模型训练与部署篇 10.1 静态内存规划：权重 / 激活 / workspace 自写 int8 推理；测 latency / RAM；GPIO + UART 闭环 int8 算子；profile；infer shell；LED 展示 板载 MNIST int8 推理闭环可演示
静态内存
程序启动前或启动时一次性分配、大小固定的内存，推理时不再动态申请。
激活 / workspace
每层网络计算的中间结果；算子计算时需要的额外临时缓冲区。
算子
完成一次卷积或矩阵乘等基本运算的代码实现。
latency / peak RAM
单张图像推理耗时；推理过程中占用的最大内存。
softmax
把各类别得分转换成概率（各项非负且总和为 1）。
profile
测量各阶段耗时和内存占用。
