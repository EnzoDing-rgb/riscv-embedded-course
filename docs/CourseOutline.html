术语表

读「三项目递进」前可先扫一眼；各章小节表内仍有章节专属名词。

术语 说明
MQTT 物联网用的发消息协议。手机、开发板不直连，都连一个「中转站」，按主题名收发文字命令（如 home/led 发 on）。
Broker MQTT 的中转站。谁发布、谁订阅，都由它转发。实验常用 Mosquitto 软件在 PC 或板子上跑。
发布 / 订阅 发布 = 往某主题发一条消息；订阅 = 盯着某主题，有新消息就收到。实验 2：手机发布、板子订阅。
主题（topic） 消息频道名，像微信群名。例：home/led 控灯，sensor/alert 报告警。
RuyiSDK / ruyi 本课统一的 RISC-V 开发工具入口：装交叉编译器、系统镜像、查板卡支持列表。
交叉编译 在 PC 上编译出只能在开发板上跑的程序，再 SSH/SCP 拷过去运行。
GPIO 板子引脚，软件可读（按键、传感器）或可写（LED、继电器）。
DHT22 温湿度传感器。板子定时读它，得到当前温度、湿度数字。
PWM 用快速开关调节平均电压，用来控风扇转速或 LED 亮度。
阈值控制 实验 1 的逻辑：温度高于某值开风扇，低于某值关。规则写死在程序或配置文件里。
pthread / 多线程 一个程序里同时跑多段逻辑（如一线程读温度、一线程处理 MQTT），互不卡死。
时序异常检测 不看单次温度，看最近一串读数（滑动窗口）像不像「正常环境」。不像 → 判异常。综合项目的 AI 部分就是这个。
电脑上训练 / 开发板运行模型 在电脑上用小脚本训练很小的模型，导出 weights.h；开发板上的 C 程序负责读取权重、算出结果，并控制风扇和灯。不在开发板上训练。
weights.h 模型权重的 C 头文件，编译进板子程序，推理时直接读这些数字做乘法相加。
智能环境终端 本课综合大项目名字：一块板子同时管测温、控风扇、MQTT 控灯/上报，并用小模型判断环境是否异常。

三项目递进

阶段 项目 闭环 串起的知识点 一句话说明
实验 1 温度传感器 + 风扇 本地自动化：读温度 → 判断阈值 → 控制风扇转速/开关 RuyiSDK 环境、C 编译部署、GPIO/外设读取、条件控制 按配置文件里的温度上下限控风扇（写好的规则，不用模型）：超过上限开风扇，低于下限关风扇。
实验 2 LED + Mosquitto + 手机 MQTT App 远程控制：手机发消息 → Broker → 开发板 → LED 亮灭 TCP/IP、MQTT 订阅发布、GPIO 输出、与实验 1 独立可演示 手机发指令，远程开关板子上的 LED。
综合项目 智能环境终端（含时序异常检测） 整合温控 + 远程控灯 + 日志配置 + 多线程；模型判「环境异常」→ 风扇/LED/MQTT 联动 ch01–ch06 全部模块；电脑上训练小模型；开发板 C 程序运行模型；可选纯规则或规则+模型 把实验 1、2 合成一个监控盒；板载小模型根据最近温度走势判「环境异常」时，板上 LED 亮红灯、风扇加档，手机 MQTT App 收到一条告警消息——三项同时发生。

硬课规

  • 板载应用必须以 C 语言 实现（MQTT 可用 libmosquitto 等 C 库）
  • 必须使用 RuyiSDK 获取工具链与目标环境
  • 必须包含 交叉编译 + Makefile（或等价工程模板） 全流程
  • 必须有 综合大项目 可演示成果，不能只做分散实验
  • 模型：PC 端轻量训练（跟课程脚本）→ 板端 C 推理；不要求板上训练
  • OpenCV、YOLO、语音 KWS、Buildroot 等 不进主线，见 docs/archive/ 选修草稿

训练与推理分工

  • PC（轻量）：采集/导入正常温湿度序列 → 提取窗口特征 → 训练 tiny MLP（或浅层模型）→ 导出 weights.h + golden 向量
  • 开发板（主线）：滑动窗口 → C 前向推理 → 正常/异常 → 风扇 + LED + MQTT
  • 课堂重点：weights.h 如何进工程、推理如何嵌进多线程、如何与规则控制共存
