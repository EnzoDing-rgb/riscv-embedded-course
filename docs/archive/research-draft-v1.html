知识调研

1. 「语音关键词控灯」是什么？难不难？

是什么
不是 ChatGPT，不是语音识别转文字。是只听几个固定词（如「开」「关」「闪」），用小神经网络判断「当前说的是哪个词」，然后控 LED。工业里叫 keyword spotting / 唤醒词，课上统一说语音关键词控灯即可。
难不难
中等，比「联网 HTTP 状态页」更炫，比「大模型 Agent」简单得多。难点在：① 麦克风接线与采音 ② 每人录一点训练样本 ③ 把训练好的小模型嵌进 C 程序。算法本身有 MLPerf Tiny、Google Speech Commands 等现成参考，不必从零推公式。
训练放哪
贵 RISC-V 板可以用 CPU + Python（numpy/PyTorch） 训 tiny 模型，认知没错；就是比 x86 慢。也可 PC 训练 → 导出权重 → 板上只推理，课时更稳。调研建议：板上推理必选，训练二选一写进课规。

2. 老师为什么提 OpenCV？主流吗？和找工作？

OpenCV 是什么
开源计算机视觉库：读摄像头、处理图像（灰度、滤波、边缘、人脸检测等）。C++/Python 都能用，嵌入式 Linux 上很常见。
老师为何放在 ch09
ch01–07 已会 GPIO/线程/文件；加摄像头是自然的第二条传感链路（ch08 听、ch09 看）。OpenCV 是「在板子上跑视觉」的事实标准，和 Ruyi 文档里 LicheePi 4A 的 OpenCV/YOLO 示例同一脉络。
主流吗
是。 机器人、工业相机、安防、车载前视、质检里大量用 OpenCV 或与其同类的 pipeline（很多公司后来用更高级的 SDK，但课里 OpenCV 仍是入门正路）。
和就业
单独「会 OpenCV 几行 API」不够找工作；「嵌入式 Linux + 摄像头采集 + 图像预处理 + 和模型/业务对接」 是真实岗位技能（智能摄像头、边缘视觉、工控检测）。OpenCV 是这条路上的工具课，不是噱头。

3. Agent / 微信控家电（OpenClaw 等）—— 为何建议不进主课

是什么
在本机跑 Agent 网关，接微信/飞书，用自然语言让大模型调工具（如 Home Assistant）控家电。
学习成本
高：公网回调、API Key、微信插件、Agent 配置、安全配对；和「学生写 C/线程/GPIO」技能栈重叠少。
和 RISC-V 课关系
可在 x86/ARM 服务器上演示；不必绑 RISC-V 板。适合讲座或 ch11 后「行业见闻」，不适合替代 OpenCV/Buildroot 实验。

4. 硬件：贵板 vs 便宜板

区别大吗
对本课 tiny 语音模型：差别主要在编译/训练快一点、内存宽裕一点，不是必须 GPU。P550 / EBC7700 档 CPU 训小模型、跑 OpenCV 更从容。
仍需统一
全班同一套麦克风 + LED 接线 比板子贵不贵更重要；I2S 兼容性先查 support-matrix。
