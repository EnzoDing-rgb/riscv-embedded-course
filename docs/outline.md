# RISC-V 嵌入式系统与边缘推理 — 教学大纲

**学时**：8 周 × 15 小时 = 120 小时  
**结构**：第 1–4 周 **嵌入式固件基础** → 第 5–8 周 **边缘 AI 推理部署**  
**前提**：C 语言 + 计算机组成原理  
**硬件**：RISC-V 开发板或模组（班内统一最小接线）；无板可用 QEMU  
**工具**：Git、GNU Make、交叉 GCC/LLVM、OpenOCD、串口、Ripes；第 5 周起 PC 端 PyTorch

**课程性质**：**嵌入式系统工程课**（外设、驱动、实时行为、资源与功耗约束），用 RISC-V 真板实践；计组/汇编是手段，不是目的。后半在同等约束下做 TinyML 部署。

---

## 课程要回答的问题

| 问题 | 本课怎么答 |
|------|------------|
| 学嵌入式能干嘛？ | 写 **固件与驱动**：采集、串口调试、中断与任务划分、为量产和联调打基础 |
| 和计组/系统课差在哪？ | 强调 **外设时序、信号、驱动分层、实时与内存预算、功耗意识** — 不是只跑通汇编实验 |
| 和纯 AI 课差在哪？ | 模型必须 **进真板**：量化、静态内存、采样与推理并行 |
| 项目要硬核吗？ | ≥60% 自写；可接 AI，须能讲清驱动与推理边界 |
| Lab 要扎实吗？ | 手写 UART shell、外设驱动、int8 推理循环 |

**一句话**：培养能在 **资源受限 RISC-V 设备** 上完成 **传感采集 + 边缘推理** 的嵌入式工程师。

# 第一部分：第 1–4 周（嵌入式固件）

**本段目标**：独立完成 **bring-up → 驱动 → 采集固件**，具备嵌入式调试与集成能力。  
**通过标准**：shell + 外设采集可现场演示（可补考后进 AI 段）。

---

## 第 1 周：开发环境、内存布局与 Bring-up（15h）

### 学什么

- **嵌入式工程习惯**：Git 分支与提交；**GNU Make** 组织交叉编译（`CFLAGS`、链接脚本参与构建）  
- 交叉编译、烧录、串口日志、GDB/OpenOCD  
- C 在嵌入式里的 **内存段**：`.text` / `.rodata` / `.data` / `.bss` / 栈 — 链接脚本与 `objdump -h` 对照  
- RV64I 与调用约定；Ripes / Compiler Explorer 建立 ISA 直觉（辅助）  
- GPIO：手册寄存器点灯，不用厂商 HAL 黑盒  

### 为什么

工业固件先会 **可复现构建 + 版本管理**，再谈外设；内存段是后续缓冲、模型权重进 Flash 的前置概念。

### Lab 1

1. 搭建仓库：`Makefile` 生成 `.elf`，`make flash` 或等价烧录目标  
2. 汇编小练习 + 真板点灯  
3. 交付：启动简图 + **各内存段地址范围表**（从 map 文件抄录并解释）

---

## 第 2 周：MMIO、UART 与驱动分层（15h）

### 学什么

- `volatile`、位域操作、MMIO 读写时序注意点  
- **自写 UART 驱动**；发送/接收缓冲  
- 简易 **驱动分层**：`hal_gpio` / `drv_uart` / `app_shell` 职责划分  
- 表驱动 **shell**：固件调试与产测入口（工业常见手段）

### Lab 2

| 命令 | 行为 |
|------|------|
| `help` / `led` / `echo` / `peek` / `ms` | 同前；`peek` 带地址白名单 |

交付：分层目录 + 串口日志；简短说明各层职责。

---

## 第 3 周：中断、时间与实时行为（15h）

### 学什么

- 异常/中断、PLIC/CLINT（按 SoC 手册）  
- **实时性**：中断延迟、ISR 禁忌、与主循环/任务分工  
- 定时器：tick、超时、软定时器思路  
- **协作式多任务**（课内实现）  
- **实时基础（讲授）**：周期任务、抖动、最坏情况延迟；FreeRTOS 任务/队列在业界的角色；与协作调度对比 — **课内不移植 RTOS**  
- 看门狗（可选加分）  

> 实时系统理论（速率单调、Linux RT 扩展等）见 **选学** 慕课脉络，本课只练「ISR 短、缓冲解耦」。

### Lab 3

秒表（定时器中断）；按键消抖 + 外部中断；双任务（采集打印 + LED）；交付：时序/ISR 分工说明（半页）。

---

## 第 4 周：总线、传感与嵌入式集成（15h）

### 学什么

- **I2C 或 SPI**：时序、寻址、读传感器寄存器 — 驱动手写  
- 模拟信号链入门：ADC 采样、简单滤波（滑动平均即可）  
- **环形缓冲**（独立 `ringbuf.c/h`，可移植、可单测）— 阶段项目核心数据结构  
- 采样率与缓冲深度、溢出策略（丢最旧 / 停采）  
- **嵌入式专题（讲授 + 短练）**：  
  - 功耗：睡眠/时钟门控概念；何时关外设  
  - 可靠性：看门狗、复位原因、启动失败怎么查  
  - 量产调试：shell 命令设计、日志格式  
- **阶段项目**：W1–W3 集成

### Lab 4 + 阶段项目（二选一）

**A. 环境监测固件**：I2C 温湿度（或等价传感器）→ 缓冲 → shell 周期 `print` / `dump`；按键改间隔  
**B. 无外设传感器**：ADC 或串口注入 → 缓冲 → `dump`；强调驱动与缓冲逻辑自写  

**共同要求**：**自写 ring buffer 模块**；扩展 shell ≥2 命令；中断或多任务采集；驱动分层清晰；自写 ≥60%。

**第 4 周末检查**：演示 `dump` 缓冲内容 + 说明深度与采样率关系。

---

# 第二部分：第 5–8 周（边缘推理 on 嵌入式）

**本段目标**：在 **第 1–4 周固件** 上部署 **自写 int8 推理**，完成 AIoT 闭环。  
**参考**：MIT 6.5940 量化与 MCU 约束、[TinyML 综述](https://arxiv.org/abs/2403.19076) 系统–算法协同。

---

## 第 5 周：TinyML 与模型导出（15h）

| 主题 | 内容 |
|------|------|
| 边缘约束 | Flash/RAM/MACs 预算表；模型为 **板子** 而缩 |
| 训练 | PyTorch tiny 网（MNIST 子集 / 2 类声纹·噪声 / W4 传感二分类） |
| PTQ 入门 | per-tensor int8 → `weights.h` |
| 推理 | `ref_inference.c`，PC golden |

**Lab 5**：训练 + PTQ + ref C + 预算表。**本周不上板。**

---

## 第 6 周：量化与压缩（15h）

- per-channel PTQ、校准、逐层误差；结构化剪枝 + 再 PTQ；QAT 简介（加分）  
- 推理实现：课内 **自写 C 算子**；CMSIS-NN / muRISCV-NN 类库作 **选学对照**

**Lab 6**：Baseline / per-channel / 剪枝+PTQ；**对比表**（体积、精度、RAM、ms）+ 1 页报告。

---

## 第 7 周：固件—推理集成（15h）

- 静态内存部署；采样任务 + 推理任务（沿用 W3/W4）  
- shell：`infer`、`latency`、`quant_info`  
- 板上 RAM/延迟实测；project 原型

---

## 第 8 周：期末 AIoT Project（15h）

1. 真板 + **≥2 种自写外设驱动** + int8 推理（for 循环可指认）  
2. 多任务或中断；现场改阈值/样本有效  
3. 提交 W6 对比表 + 板上延迟 + 设计说明  
4. 答辩 5 分钟  

| 方向 | 内容 |
|------|------|
| A 关键词/异常声 | 麦/ADC → 特征 → tiny 网 |
| B 传感智能 | W4 数据 → 特征 → MLP |
| C 极简视觉 | 低分辨率 → tiny CNN |
| D 交互装置 | 传感 → 推理 → 执行器 |

**评分**：演示 30 / 集成 25 / 代码 20 / 量化与说明 15 / 创新 10

## 课程专注范围（边界说明）

本 8 周 **课上会教**：

- RISC-V 裸机：外设驱动、中断、任务划分、采集固件  
- 嵌入式惯例：驱动分层、shell bring-up、实时与缓冲、功耗/看门狗 **入门**  
- 边缘推理：训练（PC）、量化/剪枝、裸机 int8 部署  

**课上不讲、放在延伸课/自学**（需要可自行选修，不占本大纲课时）：

- 完整 **嵌入式 Linux**：Buildroot、内核模块、字符设备驱动框架（可参考 Bootlin、百问网）  
- **完整 RTOS 产品级移植**（FreeRTOS+ 中间件栈）— 本课只讲模型与对比  
- 大模型训练与云端 MLOps  
- 自动 NAS / 大规模分布式训练  

**选学资源**（不考）：TinyMLedu、mcunet、micrograd、llama2.c（结构阅读）、muRISCV-NN；下节慕课对照。

## 推理在嵌入式中的位置

```text
  传感/ADC/I2C (W4 驱动)
        ↓ 环形缓冲
  采样任务 ←→ 推理任务 (W3 并发)
        ↓
  自写 int8 推理 (W7–W8)，权重在 Flash
        ↓
  shell：infer / latency / 告警 GPIO
```
